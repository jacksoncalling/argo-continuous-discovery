# Opportunity Solution Tree — Quick Reference

Based on Teresa Torres' Continuous Discovery Habits framework.

---

## Structure

```
Desired Outcome
  ├── Opportunity A (parent)
  │     ├── Opportunity A1 (child — more specific)
  │     └── Opportunity A2 (child — more specific)
  │           ├── Solution 1
  │           ├── Solution 2
  │           │     └── Assumption Test
  │           └── Solution 3
  └── Opportunity B (parent)
        └── ...
```

**The operator works at the opportunity layer only.** Solutions and experiments
are the team's creative work.

---

## Opportunity Quality Test

Ask: **"Is there more than one way to address this?"**

- Yes → it is an opportunity (keep it)
- No → it is a solution disguised as an opportunity (reframe it)

---

## Opportunity Types

| Type | Definition | Example |
|------|-----------|---------|
| **Pain Point** | Friction, frustration, failure | "I couldn't find the product I was looking for" |
| **Need** | Functional requirement | "I need to reorder before I run out" |
| **Desire** | Want beyond necessity | "I wish I could see how this looks on my skin tone" |

---

## Tree Rules

- **Parent-child:** A parent opportunity is addressed when all its children are addressed
- **Siblings must be distinct:** No overlap between opportunities at the same level
- **No vertical stacks:** A parent with only one child means the parent IS the child — collapse it
- **Customer perspective:** Frame opportunities from the customer's point of view, not the company's
  - Bad: "We need to improve onboarding"
  - Good: "New customers don't understand what the subscription includes"

---

## Prioritization (Four Dimensions)

When comparing sibling opportunities, assess:

| Dimension | Question |
|-----------|---------|
| **Opportunity sizing** | How many customers? How often? |
| **Market factors** | Competitive positioning, trends, table stakes vs. differentiator |
| **Company factors** | Strategic alignment, team capability, political capital |
| **Customer factors** | How important to them? How satisfied with current solutions? |

Do NOT include effort estimation at this stage. Effort belongs to solutions, not opportunities.

---

## Confidence Levels

| Level | Evidence |
|-------|---------|
| **Strong** | 3+ interviewees describing the same opportunity |
| **Moderate** | 2 interviewees, or 1 with corroborating behavior |
| **Emerging** | 1 interviewee, first mention |

## Vistaly Discovery Space (Tool Implementation)

If the team uses Vistaly, opportunities map to **Opportunity Cards** with these statuses:

`Identified → Not now → Later → Next → Now`

Vistaly adds **Insight Count** — the number of interview snapshots linked to an opportunity.
This is the tool equivalent of the confidence levels above:
- Emerging = 1 insight linked
- Moderate = 2-3 insights
- Strong = 4+ insights

Vistaly also tracks **Assumptions** with five categories: Desirability, Usability,
Feasibility, Viability, and Ethical/Legal. The operator's ESCALATE route maps to
ethical/legal assumptions that need immediate attention.
